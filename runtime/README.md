# Runtime 模块

## 负责什么
- 负责运行时状态管理
- 负责应用运行过程中的上下文对象与状态对象管理
- 负责支撑运行期控制

## 不负责什么
- 不负责业务规则定义
- 不负责场景识别
- 不负责审核结论生成

## 输入 / 输出
- **输入**: 运行时事件、上下文状态、服务状态
- **输出**: 状态对象、上下文对象、运行控制结果

## 目录说明
- `logs/` - 运行日志目录（bot.log、gray-evaluation.log 等）
- `locks/` - 锁文件目录（防止并发冲突）
- `persistence/` - 文件持久化数据存储（开发/测试用）
  - `reviews/` - 审核记录
  - `evaluations/` - 评估记录
  - `sessions/` - 会话记录
  - `messages/` - 消息记录
- `shadow-run/` - 影子运行数据目录

## 职责边界
本模块仅管理运行时状态和存储运行产物，不包含业务分析逻辑。
分析逻辑应归属于 `core` 模块。
