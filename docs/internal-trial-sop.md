# 内部试运行 SOP（Standard Operating Procedure）

> **版本**：v1.0  
> **创建时间**：2026-04-24  
> **适用对象**：主管、指定客服、内部测试人员  
> **状态**：小范围启用阶段

---

## 一、谁可以参与试运行

### 1.1 当前允许参与人员

| 角色 | 人数 | 职责 | 权限 |
|------|------|------|------|
| **主管** | 1 人 | 决策、观察、评估知识质量 | 可关闭开关、修改白名单 |
| **指定客服** | 1~2 人 | 参与训练、反馈问题 | 仅使用，不可修改配置 |
| **技术负责人** | 1 人 | 维护系统、观察日志、处理异常 | 可修改配置、查看日志 |

### 1.2 参与条件

**主管/技术负责人**：
- ✅ 了解知识注入基本原理
- ✅ 知道如何查看日志
- ✅ 知道如何关闭开关
- ✅ 有决策权（是否扩大范围、是否暂停）

**指定客服**：
- ✅ 经过简单培训（了解知识增强建议答案展示形式）
- ✅ 知道如何反馈问题
- ✅ 承诺遇到问题立即上报

### 1.3 不允许参与人员

- ❌ 未培训的客服
- ❌ 外部用户
- ❌ 全自动测试脚本（除非主管明确同意）

## 二、如何进入试运行

### 2.1 使用哪个 Bot

**Bot 名称**：Trainer Core Telegram Bot

**Bot 入口**：Telegram 搜索或扫码添加

### 2.2 走哪个入口

**入口命令**：
```
/start_training
```

**操作流程**：
1. 在 Telegram 中向 Bot 发送 `/start_training`
2. Bot 返回可选场景列表
3. 选择 `lanton_transfer_success_not_received`（转账成功但对方未到账）
4. Bot 发送第一轮用户消息
5. 客服回复消息
6. Bot 返回训练反馈 + 知识增强建议答案（如果命中知识）

### 2.3 哪些 Scenario 会触发知识增强

**当前唯一触发场景**：
- `lanton_transfer_success_not_received`（转账成功但对方未到账）

**不会触发的场景**：
- `register_flow`（注册流程指引）
- `lanton_sms_code`（注册收不到验证码）
- 其他任何场景

**识别方法**：
- 选择场景时，只有 `lanton_transfer_success_not_received` 可能出现知识增强建议答案
- 其他场景的反馈消息中**不会出现** `💡 知识增强建议答案` 部分

## 三、如何识别知识增强是否生效

### 3.1 方法一：查看 Telegram Bot 最终消息

**知识增强已生效**：
```
📋 *训练反馈 - 第 1 轮*

*场景：* 转账成功但对方未到账

✅ *本轮做得好的地方：*
• 未使用禁忌表达
• 回复详实充分

🔴 *本轮存在的问题：*
• 缺少关键信息: 请您、账单截图、绑定手机号、协助

💡 *下一步建议：*
• 确保回复包含所有必要信息，避免遗漏关键步骤。

🔄 *训练状态：* 继续训练

━━━━━━━━━━━━━━━━
💡 *知识增强建议答案*（内部试运行）  👈 看到这个 = 知识增强已生效

您好，请提供转账时间、金额和交易哈希，我们马上为您核查到账状态。

_评分：65分_
```

**知识增强未生效**（正常回退）：
```
📋 *训练反馈 - 第 1 轮*

*场景：* 注册流程指引

✅ *本轮做得好的地方：*
• 未使用禁忌表达
• 完成期望动作: 礼貌问候

🔴 *本轮存在的问题：*
• 建议包含: 谢谢

💡 *下一步建议：*
• 需要改进：建议包含: 谢谢

🔄 *训练状态：* 继续训练
_请继续回复用户消息_

（没有知识增强建议答案部分）  👈 没看到 = 未命中知识或回退（正常）
```

### 3.2 方法二：查看日志

**查看命令**：
```bash
# 实时查看知识注入日志
tail -f runtime/logs/bot.log | grep KnowledgeInjectionTrial
```

**知识增强已生效**（命中日志）：
```json
[KnowledgeInjectionTrial] 命中内部试运行: trial_hit
[KnowledgeInjectionTrial] 命中日志: {
  "timestamp":"2026-04-24T11:48:05.890Z",
  "entry_source":"training",
  "scenario":"lanton_transfer_success_not_received",
  "knowledge_hit_count":1,  👈 命中知识条数 >= 1
  "knowledge_ids":[69],
  "injected":true,  👈 注入成功
  "fallback":false,  👈 未回退
  "success":true  👈 整体成功
}
```

**知识增强未生效**（未命中）：
```json
[KnowledgeInjectionTrial] 未命中内部试运行: scenario_not_whitelisted, context: {"entrySource":"training","scenario":"register_flow","projectId":"default"}
```

**知识增强未生效**（回退）：
```json
[KnowledgeInjectionTrial] 知识注入失败，回退到原逻辑: qwen_api_timeout
[KnowledgeInjectionTrial] 回退日志: {
  "timestamp":"...",
  "injected":false,  👈 注入失败
  "fallback":true,  👈 触发回退
  "fallback_reason":"qwen_api_timeout"
}
```

### 3.3 关键指标速查

| 指标 | 在哪里看 | 正常值 | 异常值 |
|------|---------|--------|--------|
| `knowledge_hit_count` | 日志命中日志 | `>= 1` | `0`（应命中但未命中） |
| `injected` | 日志命中日志 | `true` | `false` |
| `fallback` | 日志命中/回退日志 | `false` | `true`（频繁回退需关注） |
| 知识增强建议答案 | Telegram Bot 消息 | 有（白名单场景） | 无（非白名单或回退） |

## 四、如何记录问题

### 4.1 问题类型分类

| 问题类型 | 说明 | 示例 | 严重程度 |
|---------|------|------|---------|
| **命中错知识** | 检索到的知识与场景无关或过时 | 用户问转账，返回注册 FAQ | 🔴 高 |
| **建议答案偏题** | Qwen3 生成的建议答案与场景不匹配 | 转账场景返回下载指引 | 🔴 高 |
| **触发 risky_suggested_reply** | Qwen3 标记建议答案有风险 | 建议答案包含禁忌表达 | 🔴 高 |
| **应命中但没命中** | 白名单场景但知识检索返回空 | 转账场景 knowledge_hit_count = 0 | 🟡 中 |
| **建议答案质量低** | 建议答案可用但不够专业 | 缺少关键信息、语气生硬 | 🟡 中 |
| **耗时过长** | 知识注入耗时 > 30 秒 | duration_ms > 30000 | 🟢 低 |

### 4.2 问题记录模板

**主管/客服反馈格式**：
```
【问题反馈】
- 时间：2026-04-24 19:30
- 场景：lanton_transfer_success_not_received
- 用户消息：我这边显示转账成功了，但是对方一直没收到钱
- 客服回复：您好！我们已通过官方系统查询，转账状态显示为成功...
- 知识增强建议答案：您好，请提供转账时间、金额和交易哈希...
- 问题类型：命中错知识 / 建议答案偏题 / 应命中但没命中 / ...
- 问题描述：（详细描述问题）
- 建议改进：（如有）
```

**技术负责人日志记录格式**：
```
【技术日志】
- 时间：2026-04-24 19:30
- Session ID：8fd2e142-cbd4-4aca-b9a3-8f1c6d2f93fb
- 场景：lanton_transfer_success_not_received
- 知识命中日志：（粘贴 JSON 日志）
- 问题类型：命中错知识 / 建议答案偏题 / ...
- 根因分析：（如已定位）
- 处理动作：（如已处理）
```

### 4.3 问题反馈渠道

**紧急问题**（🔴 高严重程度）：
- 立即上报主管
- 主管决定是否关闭开关
- 技术负责人立即查看日志

**一般问题**（🟡 中 / 🟢 低严重程度）：
- 记录到问题跟踪表（Excel / Notion / 飞书文档）
- 每日汇总给主管
- 技术负责人定期分析

## 五、如何处理异常

### 5.1 异常情况分类

| 异常情况 | 处理动作 | 负责人 |
|---------|---------|--------|
| **频繁命中错知识**（> 20%） | 立即关闭总开关 → 清理错知识 → 重新启用 | 主管 + 技术负责人 |
| **建议答案严重偏题**（> 30%） | 立即关闭总开关 → 检查知识质量 → 调整 prompt → 重新启用 | 技术负责人 |
| **触发 risky_suggested_reply** | 立即关闭该场景 → 检查知识内容 → 标记风险知识 | 技术负责人 |
| **频繁回退**（> 50%） | 检查 Qwen3 API 状态 → 检查知识库连接 → 修复后重新启用 | 技术负责人 |
| **系统性能严重下降** | 临时关闭知识注入 → 排查性能瓶颈 → 优化后重新启用 | 技术负责人 |
| **客服反馈强烈不满** | 暂停试运行 → 收集反馈 → 改进后重新启用 | 主管 |

### 5.2 紧急关闭流程

**场景一：一键关闭所有知识注入**
```bash
# 1. 关闭总开关
export KNOWLEDGE_INJECTION_ENABLED=false

# 2. 重启服务
npm restart

# 3. 验证已关闭
tail -f runtime/logs/bot.log | grep KnowledgeInjectionTrial
# 应看到：未命中内部试运行: trial_disabled
```

**场景二：只关闭某个场景**
```bash
# 1. 从场景白名单中移除该场景
export KNOWLEDGE_INJECTION_SCENARIO_WHITELIST=other_scenario

# 2. 重启服务
npm restart

# 3. 验证该场景已不触发
# 选择该场景进行训练，应看不到知识增强建议答案
```

**场景三：只保留日志、不输出建议答案**
```bash
# 1. 关闭总开关
export KNOWLEDGE_INJECTION_ENABLED=false

# 2. 重启服务
npm restart

# 3. 继续观察日志
tail -f runtime/logs/bot.log | grep KnowledgeInjectionTrial
# 仍会记录未命中原因，但不输出知识增强建议答案
```

### 5.3 恢复流程

**问题解决后重新启用**：
```bash
# 1. 确认问题已解决
# - 如清理了错知识，确认知识库已更新
# - 如修复了 API 超时，确认 Qwen3 服务正常

# 2. 重新开启总开关
export KNOWLEDGE_INJECTION_ENABLED=true

# 3. 重启服务
npm restart

# 4. 小范围验证（1~2 次训练）
# - 选择白名单场景
# - 确认知识增强建议答案正常输出
# - 确认日志正常

# 5. 观察 30 分钟
# - 无异常则恢复正常使用
# - 有异常则再次关闭
```

## 六、日常观察与维护

### 6.1 每日观察清单

**主管**：
- [ ] 查看今日知识注入命中次数（日志统计）
- [ ] 查看客服反馈的问题记录
- [ ] 评估建议答案质量（主观认可次数）
- [ ] 决定是否继续/暂停/扩大范围

**技术负责人**：
- [ ] 检查日志是否有异常（频繁回退、耗时过长）
- [ ] 检查知识库连接是否正常
- [ ] 检查 Qwen3 API 状态
- [ ] 处理客服反馈的技术问题

**指定客服**：
- [ ] 记录使用中遇到的问题
- [ ] 反馈建议答案质量（认可/不认可）
- [ ] 上报异常情况

### 6.2 每周总结

**每周必须输出**：
- 总样本数（本周训练次数）
- 知识命中次数（knowledge_hit_count > 0 的次数）
- 知识注入成功次数（injected = true 的次数）
- 回退次数（fallback = true 的次数）
- 主管主观认可次数（建议答案可用的次数）
- 命中错知识次数
- 应命中未命中次数

**判断是否继续试运行**：
- ✅ 继续：知识命中准确率 > 80%，建议答案认可率 > 70%，回退率 < 20%
- ⚠️ 观察：指标在临界值附近，需再观察一周
- ❌ 暂停：指标低于阈值，需整改后重新启用

### 6.3 日志统计命令


## 七、培训材料（新参与人员必读）

### 7.1 5 分钟快速培训

**什么是知识增强建议答案？**
- 系统会根据对话内容，从知识库中检索相关 FAQ
- Qwen3 基于检索到的知识，生成一个建议答案
- 建议答案会附加在训练反馈后面，供客服参考

**如何识别知识增强是否生效？**
- 看到 `💡 知识增强建议答案（内部试运行）` = 生效
- 没有该部分 = 未命中知识或回退（正常）

**遇到问题怎么办？**
- 建议答案不准确 → 记录问题，反馈给主管
- 系统异常（卡住、报错） → 立即上报
- 不确定是否正常 → 截图 + 上报

**重要提醒**：
- ⚠️ 知识增强建议答案仅供参考，不代替你的判断
- ⚠️ 如发现建议答案有误，仍按原流程回复
- ⚠️ 遇到问题立即反馈，不要自行修改配置

### 7.2 常见问题 FAQ

**Q1：为什么有时有知识增强建议答案，有时没有？**
A：只有白名单场景（当前只有 `lanton_transfer_success_not_received`）才会触发知识增强。其他场景不会触发，这是正常行为。

**Q2：知识增强建议答案一定准确吗？**
A：不一定。知识增强处于试运行阶段，建议答案可能不准确、偏题或过时。请务必自行判断，不要盲目采纳。

**Q3：发现建议答案有误怎么办？**
A：记录问题（时间、场景、用户消息、建议答案、问题描述），反馈给主管。不要自行修改系统配置。

**Q4：系统卡住或报错怎么办？**
A：立即截图上报。可能是知识注入耗时过长或触发异常，技术负责人会处理。

**Q5：可以扩大使用范围吗？**
A：需主管评估后决定。当前处于小范围试运行阶段，不允许自行扩大范围。
