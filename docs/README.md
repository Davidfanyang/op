# Docs 目录 - 项目文档

## 负责什么
- 负责项目说明文档、执行文档、接口说明、规则说明
- 负责承接开发过程中的说明类产物
- 负责对项目边界、流程、规范进行文字化沉淀

## 不负责什么
- 不负责程序运行
- 不负责数据处理
- 不负责业务判断

## 输入 / 输出
- **输入**: 项目说明、执行规范、接口定义、流程材料
- **输出**: 文档文件、说明稿、执行单

- `RED_LINES.md` - 红线规则

- `conversations-analysis-report.md` - 会话分析报告
- `conversations-integration-report.md` - 会话集成报告

## 文档维护原则
1. 每个文档应有明确的受众和用途
2. 避免重复文档，保持单一信息源
3. 代码变更后应及时更新相关文档
4. 过期文档应移动到 `archive/` 目录
