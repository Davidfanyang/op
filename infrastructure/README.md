# Infrastructure 模块

## 负责什么
- 负责数据库连接、外部服务连接、底层资源接入
- 负责支撑系统运行的基础设施能力
- 负责底层依赖对象的初始化与提供

## 不负责什么
- 不负责业务流程编排
- 不负责分析判断
- 不负责训练反馈输出

## 输入 / 输出
- **输入**: 连接配置、依赖参数、基础设施配置
- **输出**: 数据库连接、客户端实例、底层资源对象

## 子目录说明
- `persistence/file/` - 文件持久化实现（开发/测试用）
- `persistence/mysql/` - MySQL 持久化实现（生产用）

## 持久化层说明
本模块提供两种持久化实现：
- **File 实现**: 用于本地开发和测试，数据存储在 `runtime/persistence/` 目录
- **MySQL 实现**: 用于生产环境，通过连接池管理数据库连接

所有 Repository 层模块应通过本模块获取数据库连接，不直接操作底层驱动。
